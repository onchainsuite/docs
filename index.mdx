## Start Here

Onchain Suite is a retention automation platform built for Web3. It gives growth and lifecycle teams one operating loop for behavior-triggered messaging: detect onchain activity, resolve identity, segment users, message in the right channel, and measure the outcome.

Use these docs in two ways:

- If you operate campaigns, start with `Getting Started`, `Audience`, `Campaigns`, and `Automation`.
- If you build integrations, jump to `Build on the Platform` for API and webhook guidance.

## Core Loop

Onchain Suite replaces the Web3 patchwork stack with one operating loop:

1. Detect onchain and product behavior.
2. Resolve wallet-linked identity and reachable channels.
3. Build audiences from rules, reports, or SQL.
4. Send in-app and email messages through one workflow model.
5. Measure engagement, retention, and follow-up opportunity.

## Product Pillars

- Wallet-first identity keeps the wallet as the canonical user record.
- Multi-channel activation lets one trigger fan out to the channels a user can actually receive.
- Protocol plays help non-technical teams launch sophisticated retention workflows quickly.
- Intelligence tools turn contract activity and campaign response into actionable segments.

## Recommended First Steps

1. Review the platform overview and workspace model.
2. Configure sender, domain, and branding settings.
3. Connect the contract, wallet, or product events needed for your first workflow.
4. Create a small test segment, campaign, or automation and validate it end to end.
