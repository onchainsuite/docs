Campaign content is written once and rendered per recipient. This page covers the personalization syntax — the part most likely to break a send if you get it wrong.

## Merge variables

Reference a value with double braces:

```text
Hey {{ profile.greetingName }}, your wallet {{ profile.walletShort }} just qualified.
```

### Identity

| Variable | Value |
| --- | --- |
| `profile.email` | Email address |
| `profile.firstName` / `profile.lastName` | Name parts |
| `profile.fullName` | Both, when available |
| `profile.displayName` | Best available name, falling back through ENS, short wallet, then email local-part |
| `profile.greetingName` | Same, but falls back to **`there`** so it's never blank |
| `profile.initials` | Derived initials |

<Tip>
Use `profile.greetingName` in salutations. It's the only name variable guaranteed non-empty, so `Hey {{ profile.greetingName }},` can never render as `Hey ,`.
</Tip>

### Wallet and chain

| Variable | Value |
| --- | --- |
| `profile.walletAddress` / `profile.wallet` | Full address |
| `profile.walletShort` | Truncated, e.g. `0x1234…abcd` |
| `profile.ensName` | ENS name when known |
| `profile.ensOrWallet` | ENS, else short wallet, else address |
| `profile.chain` / `profile.primaryChain` | Primary chain |

### Campaign and links

| Variable | Value |
| --- | --- |
| `campaign.name` | Campaign name |
| `campaign.id` | Campaign ID |
| `links.unsubscribeUrl` | Unsubscribe link |
| `links.preferenceCenterUrl` | Preference centre link |
| `now.iso` | Render timestamp |

### Short aliases

Common variables have flat aliases that ignore case, underscores, and hyphens — so `{{first_name}}`, `{{FirstName}}`, and `{{firstname}}` are all the same:

`firstName`, `lastName`, `fullName`, `displayName`, `greetingName`, `initials`, `email`, `wallet`, `walletAddress`, `walletShort`, `ensName`, `ensOrWallet`, `chain`, `unsubscribeUrl`, `preferenceCenterUrl`.

### Contact metadata

Anything you've stored on a contact — including via [custom event attributes](/integrations/custom-events) — is available:

```text
You've made {{ txCount30d }} transactions this month.
```

Recognized metadata aliases include `engagementScore`, `holdingsCount`, `topCollection`, `portfolioValueUsd`, `txCount30d`, `volume90dUsd`, `lastActiveDaysAgo`, `claimUrl`, `referralCode`, and `loyaltyPoints`.

## Filters

Chain filters with `|`:

```text
{{ profile.firstName | default:"there" | upper }}
{{ portfolioValueUsd | number:2 }}
{{ profile.email | urlencode }}
```

| Filter | Effect |
| --- | --- |
| `default:"…"` | Fallback when the value is blank |
| `upper` / `lower` / `trim` | Case and whitespace |
| `number` / `number:2` | Thousands separators, optional decimals |
| `fixed:2` | Fixed decimal places |
| `urlencode` | URL-safe encoding |
| `json` | Serialize an object |
| `raw` | Skip HTML escaping |

Values are HTML-escaped in the body by default. Use `raw` only for content you control — never for anything a user supplied.

Unknown filters pass the value through untouched rather than erroring, so a typo like `| uppercase` silently does nothing. Check rendered output rather than trusting the syntax.

## Conditionals

```text
{{#if profile.ensName}}
  Welcome back, {{ profile.ensName }}.
{{else}}
  Welcome back.
{{/if}}
```

`{{#unless}}` is the inverse. Blocks nest. Empty arrays count as false, so you can test a list directly.

## Missing values block sends

Any variable that renders blank **without** a `default` filter is recorded as a missing field, and missing fields fail validation at launch with `RENDER_FIELD_MISSING`.

<Warning>
This is the most common reason a campaign won't launch. A variable that looks fine in preview — because *your* test contact has the value — will block the send if any recipient lacks it.

Give every optional variable a fallback:

```text
{{ profile.firstName | default:"there" }}
```
</Warning>

Unsubscribe and preference-centre variables are exempt, because they only resolve during a real send.

## Sender identity

The sender is resolved at render time, not taken from what you type. Only **verified** sender identities can be used. If the requested sender isn't verified, it falls back to your workspace default, then the first verified identity, then the platform sender — each substitution adding a warning to the render.

If test emails arrive from an address you didn't choose, check [Sender Verification](/settings/sender-verification) rather than the campaign.

## Plain text

A plain-text version is derived automatically from your HTML when you don't supply one. Supply your own when the automatic version reads poorly — stripped HTML rarely produces good copy for content-heavy emails.

## Writing checklist

- Subject line matches the intent that triggered the send
- Preview text supports the primary action rather than repeating the subject
- One main call to action
- Every optional variable has a `default` fallback
- Links are absolute, and any UTM parameters you care about are set deliberately
- Content leads with the user-relevant event, explains why it matters now, and gives one next action
